* Loomacs

Loomacs 是一套面向 macOS、由 Org 驱动的模块化 Emacs 配置框架。人工维护
=config.org=、=profiles/*.org= 与 =modules/**/*.org=；显式构建把它们编织成经过
验证的 =generated/*.el=，正常启动只离线加载生成物。

- Author / Maintainer: *SSW*
- GitHub: [[https://github.com/auspicious-ssw][@auspicious-ssw]]
- Project: [[https://github.com/auspicious-ssw/loomacs][auspicious-ssw/loomacs]]
- Framework namespace: =loomacs-=
- Current version: =0.1.0 (Unreleased)=
- Verified runtime: GNU Emacs 30.2（=/Applications/Emacs.app=）

第三方插件仍归各自项目与贡献者所有；作者信息指 Loomacs 框架、配置代码和文档。

** 当前能力

- macOS 最大化普通窗口，保留菜单栏和 Dock。
- 17pt Fira Code，中文使用 PingFang SC，图标使用 Symbols Nerd Font。
- 新顶层 GUI frame 通过 SIS + macism 切到 ABC 英文；之后尊重手动输入法选择。
- Batppuccin Mocha 是唯一配置主题；缺包或加载失败时保留 Emacs 默认外观。
- Dashboard 显示真实 Recent Files、Projects 和 Bookmarks，键盘焦点整项高亮。
- Doom Modeline 显示项目、Git、诊断和 LSP 等当前有效信息。
- Magit + Transient Posframe 提供完整 Git 工作流与 GUI 悬浮命令卡片。
- Org Modern + Org Appear，以及 Agenda、Capture 和源码块编辑。
- Vertico、Orderless、Marginalia、Consult、Embark、Corfu、Cape 补全体系。
- 内置 Eglot、xref、Flymake 和 Tree-sitter 代码能力。
- gptel AI 对话、改写和显式上下文；默认禁用媒体跟踪和工具调用。

** 启动与构建边界

#+begin_example
early-init.el
  → init.el
    → framework/loomacs-bootstrap.el
      → $XDG_STATE_HOME/loomacs/releases/current/loomacs-manifest.el
        → 无本机 Release 时回退到 generated/loomacs-manifest.el
          → 按 Manifest 的绝对路径离线加载 52 个配置单元
#+end_example

正常启动不会：

- 解析或 Tangle Org；
- 扫描 =modules/=；
- 刷新包索引、安装或升级插件；
- 下载 Tree-sitter grammar；
- 启动不存在的 Language Server；
- 读取或发送 AI 凭据；
- 自动发送文件、buffer 或其他上下文。

** 安装

#+begin_src sh
git clone https://github.com/auspicious-ssw/loomacs.git ~/.config/emacs
#+end_src

仓库跟踪 last-known-good =generated/=，因此克隆后即使缺包或离线也能进入基础
Emacs。首次启动后显式安装 Manifest 声明的第三方包：

#+begin_example
M-x package-install-selected-packages
#+end_example

也可以从终端使用明确会联网的维护命令：

#+begin_src sh
./scripts/install-packages
#+end_src

macOS 系统依赖：

#+begin_src sh
brew install --cask font-symbols-only-nerd-font
brew install laishulu/homebrew/macism
#+end_src

插件索引分别来自 GNU ELPA、NonGNU ELPA 和 MELPA，显式刷新时代理出现多个 HTTPS
连接属于三个独立软件源，不是 Emacs 启动循环联网。

** 日常修改

只编辑 Org 源码，不直接编辑 =generated/*.el=：

#+begin_src sh
./scripts/build
./scripts/check
git diff --check
#+end_src

=scripts/build= 在仓库同一文件系统的 staging 目录中执行 Tangle、Manifest 依赖检查、
Elisp 读取检查和 feature 检查。全部通过后才替换 =generated/=、创建不可变 Release
并切换 =current/previous=；失败不会覆盖现有可启动版本。

常用维护命令：

| 命令 | 用途 | 是否联网 |
|---+---+---|
| =./scripts/bootstrap= | 准备私有状态目录并运行静态诊断 | 否 |
| =./scripts/install-packages= | 刷新三个软件源并只安装 Manifest 缺包 | *是* |
| =./scripts/build= | 构建并发布新 Release | 否 |
| =./scripts/check= | 一致性、ERT、编译、离线启动检查 | 否 |
| =./scripts/doctor= | 包、字体、工具、凭据、生成物和 Release 诊断 | 否 |
| =./scripts/rollback= | 交换 current 与 previous | 否 |

** AI 使用

gptel 不在仓库中保存 API Key，也不固定可能变化的模型。推荐把凭据放进用户自行
维护的 =~/.authinfo.gpg=：

#+begin_example
machine api.openai.com login apikey password YOUR_API_KEY
#+end_example

然后使用 =C-c i m= 打开 gptel 菜单选择后端和模型，=C-c i c= 新建 Org 对话，
=C-c i s= 显式发送。未选中的文件和 buffer 不会自动成为上下文；自动化验证不会
发送真实 API 请求或消耗额度。

** 本机数据边界

#+begin_example
$XDG_STATE_HOME/loomacs/
├── releases/              # current、previous 与不可变 Release
├── logs/                  # 构建和诊断日志
└── state/                 # recentf、bookmarks、history、places、projects

$XDG_CACHE_HOME/loomacs/
├── editor/                # backup、autosave
├── build/                 # 可重建构建缓存
└── tree-sitter/           # 显式安装的 grammar
#+end_example

状态、缓存、插件源码、编译产物与凭据均不提交 Git。旧版
=${XDG_STATE_HOME}/emacs/= 中的真实状态只在新文件不存在时复制一次，旧文件保留，
不会覆盖新状态。

** 回滚

配置运行版本回滚：

#+begin_src sh
./scripts/rollback
#+end_src

该命令不修改 Org 源码、Git 工作区、插件或用户状态。源码历史回滚继续使用可追踪的
=git revert=，不要使用会覆盖未提交内容的破坏性 reset。
