#+title: Loomacs 架构
#+author: SSW

* 目标与约束

Loomacs 把“可阅读的 Org 配置源码”和“可离线启动的 Elisp 运行产物”明确分开：

1. Org 是唯一人工维护的配置事实源。
2. 构建是显式维护动作，不属于启动流程。
3. 运行时只加载已验证 Manifest 的精确文件，不扫描目录。
4. 插件安装、Tree-sitter grammar、Language Server 和 AI 请求都不隐式发生。
5. 构建失败不得替换 last-known-good。
6. 配置回滚不回滚 recentf、bookmark、文档或其他用户状态。

* 公共命名空间

Loomacs 使用四层命名边界：

| 层级 | 规则 | 示例 |
|---+---+---|
| Lisp 公共符号 | =loomacs-= | =loomacs-build= |
| 模块符号 | =loomacs-模块-= | =loomacs-dashboard-setup= |
| 私有函数 | 双连字符 | =loomacs-dashboard--update-selection= |
| Feature/生成文件 | =loomacs-= | =loomacs-dashboard=、=loomacs-dashboard.el= |
| Manifest 模块 ID | 无前缀内部数据 | =dashboard= |
| 第三方插件 | 保持上游名字 | =dashboard=、=magit=、=gptel= |

=framework/loomacs.el= 唯一拥有顶层 =defgroup=、版本和最低已验证 Emacs 版本。
模块不得定义裸全局符号，也不得 =provide 'org=、='dashboard= 等上游 feature。

每个领域目录至少包含一个总 Org 和多个子 Org。总 Org 只负责领域目标、包清单、
确定性顺序和跨插件整合；每个插件或独立能力由自己的 Org 唯一拥有。

* Manifest

=config.org= 只生成一个普通 plist，不定义宏 DSL：

#+begin_src emacs-lisp
'(:format 1
  :profile macos
  :requirements (...)
  :modules
  ((:id dashboard
    :feature loomacs-dashboard
    :source "modules/ui/dashboard.org"
    :output "ui/loomacs-dashboard.el"
    :requires (ui batppuccin nerd-icons project recentf bookmark)
    :packages (dashboard)
    :critical nil)))
#+end_src

构建阶段验证必填字段、唯一 ID/feature/source/output、安全相对路径、未知依赖和
循环依赖，再使用稳定拓扑排序固化加载顺序。运行时仍复核所有生成文件的 Lisp
结构和 =provide=，但不重新解释依赖意图或扫描源码目录。

* 构建数据流

#+begin_example
config.org
  └─显式 target Tangle→ staging/generated/loomacs-manifest.el
       └─解析并验证依赖图
            └─逐个 Org 显式 target Tangle 到 staging
                 ├─验证输出没有越出 staging
                 ├─完整读取每个 Elisp 文件
                 ├─核对 feature
                 └─记录全部源码 SHA-256
                      └─替换仓库 generated/
                           └─复制为本机不可变 Release
                                └─切换 current/previous
#+end_example

Org 源文件禁止写 =:tangle yes= 或具体路径。实测这些参数会覆盖 builder 的 staging
目标并写回源码旁；=loomacs-tangle-file= 会在产生任何输出前拒绝这种配置，并在
Tangle 后再次核对 Org 返回路径。

* 启动数据流

#+begin_example
early-init.el
  ├─临时 GC 放宽
  ├─禁用 package-enable-at-startup
  └─最大化普通 frame

init.el
  └─绝对路径加载 framework/loomacs-bootstrap.el
       ├─优先选择 $XDG_STATE_HOME/loomacs/releases/current/
       ├─没有 current 时选择仓库 generated/
       └─读取 loomacs-manifest.el
            └─按生成路径依次 load 模块并核对 feature
#+end_example

启动路径不加载 =loomacs-build.el= 或 =loomacs-tangle.el=，因此不会因为 Org 外观、
保存 hook、网络或构建工具而改变运行行为。

* 状态与缓存

- 仓库：可复现源码、生成物、锁文件、文档和测试。
- =$XDG_STATE_HOME/loomacs/state/=：history、recentf、projects、bookmarks、places、custom。
- =$XDG_STATE_HOME/loomacs/releases/=：本机 current、previous 和不可变 Release。
- =$XDG_STATE_HOME/loomacs/logs/=：构建与诊断日志。
- =$XDG_CACHE_HOME/loomacs/editor/=：backup 和 autosave。
- =$XDG_CACHE_HOME/loomacs/tree-sitter/=：显式安装、可重建的 grammar。
- =~/.authinfo.gpg= 或系统凭据存储：AI 密钥；永不进入仓库和日志。

* 已借鉴而未复制的设计

- [[https://github.com/doomemacs/modules/tree/main/modules][Doom Emacs modules]]：类别/模块目录和贴近模块的文档契约。
- [[https://github.com/SystemCrafters/crafted-emacs/tree/master/modules][Crafted Emacs modules]]：原生、显式的 Emacs Lisp 模块思路。

Loomacs 不复制 Doom 的宏、CLI、autoload 生成器或包管理器，也不把个人配置伪装
成 ELPA 插件。第一版不支持热重载；发布新配置后正常重启 Emacs，边界更可靠。
