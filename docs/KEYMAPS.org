#+title: Loomacs 快捷键与冲突审计
#+author: SSW

* 修饰键约定

| 物理按键 | Emacs 逻辑修饰键 | 说明 |
|---+---+---|
| Caps Lock | Control | 用户明确习惯，Loomacs 不修改 |
| Option | Meta | =M-x=、=M-f=、=M-b= 等核心 Emacs 语义 |
| Command | Super | 保留给 macOS/未来专用组合 |

本文所有 =C-= 都指物理 Caps Lock 映射后的逻辑 Control。=C-g= 必须始终是
=keyboard-quit=；Loomacs 不为任何插件重绑定它。

* macOS 系统级冲突

检查基线：2026-07-18。系统或其他应用的 global hotkey 会在按键到达 Emacs 前拦截，
Elisp 无法靠“更高优先级”覆盖它，因此必须分别记录。

** 已处理

| Emacs 键 | 核心语义 | 原外部占用 | 当前处理 |
|---+---+---+---|
| =C-SPC= | set-mark-command | macOS 输入源 | Apple Symbolic HotKey 60 已停用 |
| =C-M-SPC= | mark-sexp | macOS 输入源 | Apple Symbolic HotKey 61 已停用 |
| =C-g= | keyboard-quit | Chrome Gemini/Glic | 启动迁移到 =C-S-s-G=，导航迁移到 =C-S-s-N= |
| =M-SPC= | cycle-spacing | iTerm2 Hotkey | 迁移到 =C-S-s-T= |

Chrome 当前只公开启动与导航两个 Gemini 快捷键；不臆造第三个设置。iTerm2 的
system-wide Hotkey 已不占用 Option-Space。

** 明确保留给 macOS

=C-↑=、=C-↓=、=C-←=、=C-→= 继续用于调度中心、App Exposé 和空间切换。用户明确
不需要 Emacs 的这些方向键语义，因此 Loomacs：

- 不关闭对应 macOS 快捷键；
- 不增加 Emacs 替代绑定；
- 不把它们列入核心键保护失败项。

** 当前边界

- Spotlight 的 =Command-Space= 基线为关闭。
- ChatGPT/Codex 使用双击 Command，不占用单 chord。
- 未检测到 Raycast、Alfred、Karabiner、BetterTouchTool、Rectangle、Hammerspoon 或
  =skhd= 额外热键管理器。
- 裸 F1–F4、F10、F11 保留 macOS 亮度、调度中心和媒体行为，需要 Fn 才保证送达。
- 中文输入源可能先处理 Dashboard/Magit 的裸字母；Loomacs 只保证新 GUI frame 默认
  英文，不在正文编辑过程中持续抢输入源。

升级 macOS、Chrome、iTerm2 或新增全局热键软件后应重新只读审计。=scripts/check=
负责内部 Emacs keymap 漂移，不能替代系统级检查。
