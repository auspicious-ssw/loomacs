#+title: Loomacs 运维与排障
#+author: SSW

* 正常启动契约

正常启动必须满足：

1. 不解析 Org、不 Tangle、不安装包、不访问网络。
2. 从有效 =current= Release 启动；没有本机 Release 时使用 Git =generated/=。
3. 52 个模块按 Manifest 精确路径加载并提供预期 feature。
4. Batppuccin 可用时只加载 Mocha；异常时不切换备用主题。
5. GUI frame 最大化但不是 macOS 全屏空间。
6. 字体为约 17pt Fira Code，中文回退 PingFang SC。
7. 新顶层 GUI frame 切到 ABC；child-frame、TTY 和后续手动切换不被抢占。
8. Dashboard、Modeline、补全、Magit、Org、LSP 和 AI 缺包降级不能阻断基础编辑。
9. 启动完成后 GC 恢复为 16 MiB / 0.1。

* 构建

#+begin_src sh
./scripts/build
#+end_src

构建过程：

1. 取得仓库 =.loomacs-build.lock/=。
2. 在仓库同一文件系统创建 staging。
3. Tangle =config.org= 并验证 Manifest。
4. 稳定拓扑排序后逐模块 Tangle。
5. 检查越界路径、Elisp 读取和 =provide=。
6. 写入确定性源码 SHA-256 metadata。
7. 替换 =generated/=。
8. 创建本机不可变 Release 并切换 =current/previous=。

任一步失败都返回非零退出码并保留旧 generated/current。若异常退出留下构建锁，先
查看锁内 =owner.el= 的 PID，确认对应进程已经不存在，再人工清理；不要在仍有构建
进程时强行删除锁。

* 完整检查

#+begin_src sh
./scripts/check
#+end_src

检查必须覆盖：

- Shell 语法与 Git diff whitespace。
- Manifest/Org/generated 一致性。
- 7 组框架 ERT。
- framework、test 和生成模块零警告编译。
- 当前已安装包路径离线 batch 启动。
- 空包目录离线降级启动。
- 包版本/commit、字体、外部命令、Language Server 与 auth-source Doctor。
- 核心和自定义快捷键唯一性。

=scripts/check.sh= 只作为旧命令兼容入口，转发到 =scripts/check=。

* Doctor

#+begin_src sh
./scripts/doctor
#+end_src

输出级别：

- =INFO=：已验证通过或正常状态。
- =WARNING=：可选能力缺失，例如 Markdown server、Tree-sitter grammar 或 AI 凭据。
- =ERROR=：生成物过期、Manifest 无效、包缺失/漂移或 current Release 损坏。

Doctor 是只读检查，不会安装缺失项。认证检查只返回“是否存在”，不显示 secret。

* 插件安装与升级

启动后显式运行：

#+begin_example
M-x package-install-selected-packages
#+end_example

或显式执行（该命令会访问 GNU ELPA、NonGNU ELPA 与 MELPA）：

#+begin_src sh
./scripts/install-packages
#+end_src

升级必须一次处理一个直接包：

1. 明确刷新对应软件源。
2. 安装目标版本并记录实际 =package-desc=。
3. 更新 =lock/packages.el= 的版本、commit、URL 及必要传递依赖。
4. 运行 build、check、doctor。
5. 在真实 GUI 中验收受影响界面。

不要把 =package-refresh-contents=、=package-install= 或 =package-upgrade-all= 写进启动 Hook。

* Tree-sitter 与 LSP

=C-c e t= 显式选择并安装 grammar；产物进入
=$XDG_CACHE_HOME/loomacs/tree-sitter/=。启动只为已经安装且 ABI 可用的 grammar 设置
mode remap。

Eglot 只在以下条件全部满足时自动启动：

- 当前 buffer 访问本地文件；
- 文件属于 =project.el= 项目；
- 当前 major mode 有声明的 server；
- 至少一个候选 executable 实际存在。

远程文件、临时 buffer、未知项目和缺 server 都不会隐式启动进程。使用 =C-c e e=
可以手动启动；=C-c e d b/p= 查看 buffer/project 诊断。

* AI

推荐在 =~/.authinfo.gpg= 中保存：

#+begin_example
machine api.openai.com login apikey password YOUR_API_KEY
#+end_example

首次使用前通过 =C-c i m= 选择 gptel 后端和模型。Loomacs 不硬编码模型名；模型列表
属于上游和服务端的时变事实。=C-c i s= 才会发起网络请求，测试不会发送。

* 回滚

#+begin_src sh
./scripts/rollback
#+end_src

要求 current 和 previous 都是完整有效 Release。回滚成功后重启 Emacs；命令只交换
链接，不改 Git、不删除状态、不降级插件。

源码回滚使用：

#+begin_src sh
git log --oneline
git revert COMMIT
./scripts/build
./scripts/check
#+end_src

* 常见问题

** Dashboard 显示 No items

它读取真实 recentf、project.el 和 bookmark 状态，不是下载目录。先正常打开文件、
访问项目或创建书签，再在 Dashboard 按 =g=。

** Dashboard 的 r/p/m 没有移动

确认 buffer 是 =*dashboard*=，输入源为英文；=r/p/m= 只移动到相应区域并循环选择，
=RET= 才打开。当前项应整行紫色高亮，鼠标悬停不应产生第二个持久焦点。

** 没有 Batppuccin

运行 Doctor 检查包版本。包缺失时显式安装；包存在但主题加载报错时保留默认外观，
查看 =*Messages*= 和 Loomacs 日志，不能靠增加第二套主题掩盖错误。

** Magit 菜单没有悬浮

Posframe 只用于 GUI Transient 菜单。TTY、batch 或 child-frame 不可用时回退原生底部
菜单；Magit Status、diff 和 log 本来就使用普通窗口。

** 打开 Emacs 后不是英文

确认 =/opt/homebrew/bin/macism= 可执行，并检查 macOS 辅助权限。Loomacs 只在顶层
GUI frame 创建时切一次，不会持续轮询或覆盖启动后的中文选择。

** AI 提示没有 Key

这是未配置凭据时的预期结果。Loomacs 不生成或读取明文 Key；按安全边界配置
=auth-source= 后再次显式发送，不要把 Key 写进 Org、local Profile 或环境截图。

** 配置加载失败

先运行 =scripts/doctor=。如果 current 损坏而 previous 有效，运行 rollback；如果
=generated/= 与 Org 摘要不一致，保留工作区后重新 build。禁止手改 generated 或使用
=git reset --hard= 覆盖未提交内容。

* 真实 GUI 验收

=test/loomacs-gui-validation.el= 可在独立 Emacs.app 进程中检查最大化 frame、Fira Code、
PingFang SC、Dashboard、Doom Modeline、Corfu child-frame、Transient Posframe 和 ABC
输入源。它只写调用方指定的临时结果/截图并退出，不属于正常启动；自动化不能用它
代替人工查看截图。
